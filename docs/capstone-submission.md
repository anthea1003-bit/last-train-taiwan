# Kaggle Capstone Submission Pack

Last updated: 2026-06-24

## Submission Positioning

Recommended track: **Freestyle**.

Reason: the project is an interactive Taiwan travel narrative game with a
local Conductor Agent, deterministic route evaluation, bilingual content, and
security/evaluation evidence. It is not primarily a business automation
workflow or a single public-good service, so Freestyle is the clearest and
most honest category.

## One-Sentence Pitch

`最後一班不存在的環島列車` is a zero-token, bilingual Taiwan travel
adventure where a local Conductor Agent helps players reason through landmark
anomalies, collect memories, and unlock a hidden Penghu route.

## Honest Tooling Statement

The core game implementation and bug fixes were primarily completed through
Codex collaboration. Antigravity is used as a course-aligned agentic coding
surface for refinement, repository inspection, and workflow demonstration.

Use this phrasing in the writeup or video:

> The project was developed collaboratively with Codex, then refined and
> demonstrated through an Antigravity agentic coding workflow for the course
> evidence.

Do not claim that the whole game was mainly generated by Antigravity.

## Kaggle Writeup Draft

Title:

`The Last Impossible Round-Island Train: A Zero-Token Taiwan Travel Agent Game`

Suggested structure:

1. **Problem / Idea**: make the 5-Day AI Agents concepts tangible through a
   public, free, bilingual adventure game rather than a paid cloud demo.
2. **What I Built**: describe the six Taiwan landmarks, hidden Penghu seventh
   station, memory/stamp system, and local Conductor Agent.
3. **Agentic Workflow**: explain Codex collaboration plus Antigravity
   refinement, Agent Skills, and development-time MCP research.
4. **Safety and Cost Boundary**: no public runtime model calls, no API keys,
   no personal data, no backend, no billable Google Cloud resources.
5. **Evaluation**: summarize unit tests, localization integrity checks,
   non-spoiler Agent tests, mobile/desktop verification, and hidden-ending
   checks.
6. **What I Learned**: vibe coding becomes production-grade when constrained
   by specs, tests, security rules, and a clear demo path.
